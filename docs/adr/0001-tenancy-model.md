# ADR-0001: Tenancy Model

## Status

Accepted

## Context

Нужно определить модель мультитенантности для платформы PMServices.

## Decision

Выбрана модель **Organization-based tenancy**:
- Каждая организация — отдельный тенант
- Пользователь может быть членом нескольких организаций
- Данные изолированы по `org_id`

## Consequences

### Положительные

- Гибкость: пользователь работает в нескольких организациях
- Простота: одна БД, фильтрация по org_id

### Отрицательные

- Требуется внимательность к изоляции данных
- Все запросы должны учитывать org_id
