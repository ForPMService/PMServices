# Architecture Decision Records (ADR)

Документирование ключевых архитектурных решений проекта.

## Что такое ADR?

ADR — это короткий документ, фиксирующий важное архитектурное решение вместе с контекстом и последствиями.

## Формат

Используем [шаблон](./0000-template.md) на основе Michael Nygard's ADR format:

1. **Title** — краткое название решения
2. **Status** — Proposed / Accepted / Deprecated / Superseded
3. **Context** — что привело к необходимости решения
4. **Decision** — что решили
5. **Consequences** — положительные и отрицательные последствия
